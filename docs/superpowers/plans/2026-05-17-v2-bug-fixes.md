## Task 1: Fix check_login_lock crash (Critical)

**Files:**
- Modify: `accounts/utils.py:119`
- Modify: `accounts/views.py:340` (LoginView), `accounts/views.py:443` (employee_login_page), `accounts/views.py:480` (super_admin_login_page)

**Root cause:** `check_login_lock()` returns `bool`. Three callers (employee_login_page, super_admin_login_page) treat it as `dict` → `TypeError: 'bool' object is not subscriptable`. Fix: make the function return a dict everywhere and update all callers.

- [ ] **Step 1: Rewrite check_login_lock in accounts/utils.py**

Replace lines 119–130:

```python
def check_login_lock(email: str) -> dict:
    """
    Returns {'locked': bool, 'remaining_minutes': int}.
    remaining_minutes is 0 when not locked.
    """
    from .models import LoginAttempt

    try:
        attempt = LoginAttempt.objects.get(email=email)
        if attempt.is_locked:
            remaining = max(0, int((attempt.locked_until - timezone.now()).total_seconds() / 60)) + 1
            return {'locked': True, 'remaining_minutes': remaining}
        return {'locked': False, 'remaining_minutes': 0}
    except LoginAttempt.DoesNotExist:
        return {'locked': False, 'remaining_minutes': 0}
```

## Task 3: Fix back-to-search blank page / JSON error from checkout

**Files:**
- Modify: `templates/payments/checkout.html` — back button logic

**Root cause:** The "Back to search" button in checkout triggers a confirmation popup. On confirm, it navigates to `/rooms/search/page/` but the search JS on that page immediately auto-searches using URL params. Since no dates are in the URL, the search POST runs without `check_in`/`check_out` and returns a 400 JSON error displayed raw on screen.

**Fix:** Store the original search params in `sessionStorage` when the checkout page loads, and redirect back to search with those params in the URL so the search page has proper dates.

- [ ] **Step 1: In checkout.html JS — when the page loads and booking data is fetched, write the original search dates to sessionStorage**

```javascript
    sessionStorage.setItem('ttr_last_search_checkin', booking.check_in);
    sessionStorage.setItem('ttr_last_search_checkout', booking.check_out);
    sessionStorage.setItem('ttr_last_search_guests', booking.guests || 1);
}
```

- [ ] **Step 2: Update the back-to-search navigation to use sessionStorage params**

Find the back navigation click handler (the one that shows the confirmation modal). On confirm, build the redirect URL with stored params:

```javascript
function goBackToSearch() {
    const checkIn = sessionStorage.getItem('ttr_last_search_checkin') || '';
```

```javascript
function showSuggestedRooms() {
    const suggestedSection = document.getElementById('suggested-rooms');
    if (!suggestedSection) return;
    suggestedSection.classList.remove('hidden');

    // Fetch all active rooms (no date restriction)
    fetch('/rooms/search/', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'X-CSRFToken': getCsrfToken(),
        },
        body: JSON.stringify({
            // Use far-future dates to get all rooms — server excludes confirmed bookings
            check_in: getTomorrowStr(),
            check_out: getDayAfterStr(),
            guests: 1,
            sort: 'recommended',
        }),
    })
    .then(r => r.json())
    .then(data => {
        if (data.rooms && data.rooms.length > 0) {
            renderRoomCards(data.rooms, document.getElementById('suggested-rooms-grid'));
        }
    })
    .catch(() => {});
}

function getTomorrowStr() {
    const d = new Date(); d.setDate(d.getDate() + 1);
    return d.toISOString().split('T')[0];
}
function getDayAfterStr() {
    const d = new Date(); d.setDate(d.getDate() + 2);
    return d.toISOString().split('T')[0];
}
```
